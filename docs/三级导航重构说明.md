# LT导航三级导航重构说明

## 1. 本次改造目标

本轮将原项目从“单层分类推荐站”升级为“三级导航站”：

- 一级：大类
- 二级：中类
- 三级：小类
- 所有站点最终挂载到三级小类

同时新增了：

- 书签 HTML 导入
- 导航源 URL 抓取预览 / 覆盖导入
- 后台三栏工作台
- 内置书签种子数据

## 2. 数据结构

### categories

新增树形能力：

- `id`
- `name`
- `slug`
- `description`
- `parent_id`
- `level`
- `sort_order`

说明：

- `level=1` 表示大类
- `level=2` 表示中类
- `level=3` 表示小类

### sites

保留原有字段并补充：

- `category_id`
- `source_site`
- `source_category_path`
- `normalized_domain`

说明：

- 前台主要按 `category_id` 与分类树展示
- `source_site` 用于记录导航源或书签来源
- `normalized_domain` 用于后续去重和排查

## 3. 默认信息架构

当前内置的大类包括：

- AI工具
- 设计
- 产品与增长
- 出海与跨境
- 开发与开源
- 资源与素材
- 综合导航

另外补了一个更实用的兜底分支：

- 综合导航 → 通用收录 → 常用工具 / 导航源 / 待整理站点

这样即使自动分类暂时不能完美命中，也不会出现站点悬空。

## 5. 后台使用方式

后台路径：

- `/admin`

当前后台布局：

- 左侧：分类树 + 新增分类
- 中间：站点列表 + 导入区
- 右侧：新增 / 编辑当前站点

### 新增分类

建议顺序：

1. 先建一级大类
2. 再建二级中类并挂到大类
3. 最后建三级小类并挂到中类

### 新增站点

在右侧编辑区填写：

- 名称
- URL
- 三级分类
- Logo/Base64
- 简介
- 标签
- 来源站点

### 图片上传

支持本地图片直接上传：

- 选择图片后会自动转为 Base64
- 无需手工转换

## 6. 导入方式

### 方式一：内置种子导入

适合快速初始化整站：

1. 进入后台
2. 在“导入区”中选择“内置种子”
3. 先点“预览统计”
4. 再点“覆盖导入”

### 方式二：书签 HTML 导入

适合继续导入你后续导出的浏览器书签：

1. 粘贴 Netscape 书签 HTML
2. 点击“预览导入”
3. 确认无误后点击“覆盖导入”

### 方式三：导航源抓取

适合从导航站继续扩展：

1. 每行填写一个导航源 URL
2. 点击“抓取预览”
3. 需要覆盖站点时点击“抓取并覆盖”

## 7. 当前分类规则说明

自动分类采用关键词映射，优先覆盖：

- AI导航 / AI工具
- UI/UX / 设计导航 / 设计资源
- 开发工具 / API / 文档
- 出海 / 跨境 / 电商导航
- 模板 / 图标 / 素材

如果无法稳定命中，会进入：

- `综合导航 > 通用收录 > 待整理站点`

后续可以在后台手工细调。
